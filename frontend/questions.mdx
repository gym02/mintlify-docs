## 六、前端

### 1、React 生命周期（以函数组件为主）

- 挂载：首次渲染；`useEffect(() => {}, [])` 在挂载后执行。  
- 更新：state/props 变化触发重新渲染；useEffect 依赖变化后执行。  
- 卸载：组件移除；useEffect 返回的 cleanup 函数在卸载时执行。  
（类组件：componentDidMount / componentDidUpdate / componentWillUnmount）

### 2、什么是虚拟 DOM

虚拟 DOM 是对真实 DOM 的一层 JS 对象抽象。React 通过对比新旧虚拟 DOM（diff），计算出最小变更，再批量更新真实 DOM，从而减少直接操作 DOM 的成本，并带来更可预测的 UI 更新模型。

虽然我暂时没有深入用过 React，但我有 Vue 的，两者基础核心都是组件化和数据驱动，不同的是 React 使用 JSX，即 HTML 和 JS 结合起来，同时在状态管理上常用 Redux。这周周末也在看官方文档和编写简单的代码，我觉得以我的学习能力，能很快的学习并上手实际开发。

### 3、Vue3 生命周期（面试好背版）

- onMounted：组件第一次渲染到页面后执行，用来做初始化（请求数据、绑定事件、开定时器、初始化图表等）。  
- onUpdated：组件更新并完成 DOM 更新后执行；如果要拿到更新后的 DOM，配合 nextTick。  
- onUnmounted：组件销毁时做清理（移除事件监听、清定时器、关 WebSocket），避免内存泄漏和重复执行。  

一句话总结：挂载做初始化，更新后拿 DOM 用 nextTick，卸载做清理。

### 4、async 和 await 的作用（面试口语版）

**面试一句话**：async 标记“这是异步函数”，await 在函数里“等 Promise 出结果再执行下一行”，这样异步逻辑可以按顺序写，不阻塞页面，错误用 try/catch 抓。

**为啥要写 await？** 代码虽然是按顺序写的，但不写 await 时，像 `fetch()` 这种异步操作会**一调用就立刻往下执行下一行**，不会等请求回来；写 await 才是“等这一行有结果了再执行下一行”，这样**拿到结果的顺序**才和代码顺序一致。

**一句话**：Promise 是 JS 里表示“一个将来才会有的结果”的对象。比如发请求、读文件，结果要等一会儿才来，Promise 就是先占个位，等结果到了再通过 `.then()` 或 `await` 给你用。
