## 六、前端

### 1、React 生命周期（以函数组件为主）

- 挂载：首次渲染；`useEffect(() => {}, [])` 在挂载后执行。  
- 更新：state/props 变化触发重新渲染；useEffect 依赖变化后执行。  
- 卸载：组件移除；useEffect 返回的 cleanup 函数在卸载时执行。  
（类组件：componentDidMount / componentDidUpdate / componentWillUnmount）

### 2、什么是 DOM（面试口述版）

**DOM** 全称 **Document Object Model（文档对象模型）**。浏览器把 HTML 解析成一棵**树形结构**，页面里的每个标签（`div`、`p`、`button` 等）都是树上的一个**节点**；这套由浏览器维护、可以用 JavaScript 读写的树，就是 DOM。

- **它是什么**：网页在内存里的「结构化表示」，你可以用 JS 通过 DOM API 去**查节点、改文字、增删元素、绑事件**（例如 `document.querySelector`、`element.textContent`、`addEventListener`）。
- **和「你看到的页面」的关系**：用户看到的是浏览器根据 DOM（结合 CSS）**渲染**出来的界面；改 DOM，界面一般会跟着变（可能还要配合样式重算、重绘）。
- **为什么面试常提**：前端框架（Vue / React）最终还是要**更新真实 DOM**；「虚拟 DOM」则是用 JS 对象先描述一版 UI，再和真实 DOM 对应起来，减少直接乱改 DOM 带来的性能问题。

**白话先懂：** 你写的 HTML 像一份「说明书」，浏览器读完后，在内存里搭出一棵**树**——每个标签（如一个 `<div>`）就是树上的一个**小块**。这棵在内存里、能被 JS 点名修改的树，就叫 DOM。你用 JS 改某个节点（删一行字、加一个按钮），屏幕上对应位置也会变。

**面试一句话（背这句）：** **DOM = 浏览器根据 HTML 在内存里生成的那棵「页面结构树」，JS 通过 DOM 来改页面长什么样。**

### 3、什么是虚拟 DOM

虚拟 DOM 是对真实 DOM 的一层 JS 对象抽象。React 通过对比新旧虚拟 DOM（diff），计算出最小变更，再批量更新真实 DOM，从而减少直接操作 DOM 的成本，并带来更可预测的 UI 更新模型。

### 4、Vue3 生命周期（面试好背版）

- onMounted：组件第一次渲染到页面后执行，用来做初始化（请求数据、绑定事件、开定时器、初始化图表等）。  
- onUpdated：组件更新并完成 DOM 更新后执行；如果要拿到更新后的 DOM，配合 nextTick。  
- onUnmounted：组件销毁时做清理（移除事件监听、清定时器、关 WebSocket），避免内存泄漏和重复执行。  

一句话总结：挂载做初始化，更新后拿 DOM 用 nextTick，卸载做清理。

### 5、Promise 的作用是什么？（面试口述版）

**一句话：** Promise 是异步操作的一种方式，你用 **`.then` / `.catch`** 或 **`async/await`** 来接结果，**少写一层套一层的回调**。

**为啥要有它？**

- 发请求、定时器这类事**不能马上出结果**，要「等一会儿」。
- 老写法是**回调套回调**，多了很难读；Promise 用 **成功走 `.then`、失败走 `.catch`**，还能 **`.then` 链式**接下一步。
- 状态大致三种：**进行中 / 成功 / 失败**（知道名字即可：`pending`、`fulfilled`、`rejected`）。

**常见用法：**

```js
fetch('/api/user')
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => console.error(err))
```

**和 async/await 的关系：async/await 本质上是 Promise 的语法糖。await 后面必须是一个 Promise，它会暂停函数执行，等 Promise 完成，再返回结果。
### 6、async 和 await 的作用（面试口语版）

**面试一句话**：async 标记“这是异步函数”，await 在函数里“等 Promise 出结果再执行下一行”，这样异步逻辑可以按顺序写，不阻塞页面，错误用 try/catch 抓。

**为啥要写 await？** 代码虽然是按顺序写的，但不写 await 时，像 `fetch()` 这种异步操作会**一调用就立刻往下执行下一行**，不会等请求回来；写 await 才是“等这一行有结果了再执行下一行”，这样**拿到结果的顺序**才和代码顺序一致。

**和 Promise：** `await` 后面跟的一般是 **Promise**；上一节讲的 Promise 是「异步结果的容器」，async/await 是**基于 Promise 的语法糖**，让写法更直观。

### 7、浏览器输入 URL 并按回车之后，会发生什么？（面试口述版）

**面试一句话：** 解析 URL → DNS 解析得 IP → 建立 TCP 连接 → 发 HTTP 请求 → 服务器处理并返回响应 → 浏览器解析并渲染页面 → 连接关闭（或保持）。

**分步说明：**

1）**解析 URL**  
浏览器解析你输入的地址，得到协议（http/https）、域名、端口、路径、查询参数等。若是 HTTPS，会涉及后续的 TLS 握手。

2）**DNS 解析**  
通过域名查对应的 **IP 地址**。先查浏览器缓存、再查本机 hosts、再查本地 DNS、再递归/迭代查权威 DNS，得到 IP 后才会发起真正的网络请求。

3）**建立 TCP 连接（三次握手）**  
浏览器和服务器之间建立 TCP 连接：客户端发 SYN → 服务端回 SYN+ACK → 客户端再回 ACK，连接建立。若是 HTTPS，在 TCP 之上还会进行 **TLS 握手**（协商版本、交换证书、生成会话密钥等），建立加密通道。

4）**发送 HTTP 请求**  
浏览器沿已建立的连接发送 **HTTP 请求**（方法、URL、请求头、Cookie、请求体等）。服务器根据请求路径和参数处理业务，查库、拼数据等。

5）**服务器处理并返回 HTTP 响应**  
服务器返回 **状态码、响应头、响应体**（如 HTML、JSON）。常见状态码：200 成功、301/302 重定向、304 协商缓存、404 未找到、500 服务端错误。

6）**浏览器解析并渲染页面**  
- **解析 HTML** 生成 DOM 树；遇到 **CSS** 解析成 CSSOM；**JS** 可能阻塞解析或参与 DOM/CSSOM 的修改。  
- **DOM + CSSOM** 合成渲染树（Render Tree），再**布局**（Layout）算位置、**绘制**（Paint）成像素，最后**合成**（Composite）显示到屏幕。  
- 若响应里还有图片、脚本、样式等资源，浏览器会再发请求去拉取（可能复用连接），重复“请求→响应→解析”的过程。

7）**连接关闭或保持**  
HTTP/1.0 默认请求完就关 TCP；**HTTP/1.1 默认 Keep-Alive** 复用连接；HTTP/2 多路复用，一个连接上并发多个请求。

**背诵要点：** URL 解析 → DNS 拿 IP → TCP 三次握手（HTTPS 再加 TLS）→ 发 HTTP 请求 → 服务器返回响应 → 浏览器解析 HTML/CSS/JS、建 DOM、渲染树、布局、绘制 → 连接复用或关闭。

### 8、HTTP 和 HTTPS 的区别（面试口述版）

**核心区别：**

| 对比项 | HTTP | HTTPS |
|--------|------|-------|
| **传输** | 明文，内容可被窃听、篡改 | 加密，在 TLS 通道里传输，防窃听、防篡改 |
| **端口** | 默认 80 | 默认 443 |
| **身份与完整性** | 无服务器身份校验，易被冒充 | 通过**证书**校验服务器身份，并保证数据完整性 |
| **实现** | 直接跑在 TCP 上 | 在 TCP 之上加一层 **TLS/SSL**，再在加密通道里跑 HTTP |

**HTTPS 怎么保证安全？**  
- **TLS 握手**：建立连接时协商版本、交换证书、用非对称加密协商出**会话密钥**，后续用**对称加密**在通道里传数据（对称加密快，非对称只用于握手阶段）。  
- **证书**：服务器出示由 CA 签发的证书，浏览器验证证书链和域名，确认“对面是正牌服务器”，再继续；证书里含公钥，用于加密会话密钥或做密钥交换。  
- 所以：**身份**靠证书、**机密性**靠加密、**完整性**靠摘要/签名，三者一起保证安全。

**面试一句话：** HTTP 明文、80 端口；HTTPS 在 TCP 上套一层 TLS，加密传输、默认 443、用证书校验服务器，防窃听防篡改防冒充。

## Vue 相关

### 1、v-if 和 v-show（背诵版）

- **v-if**：`false` 时**不渲染**（没有 DOM），切换会**销毁/重建**；费切换、省初始。
- **v-show**：**始终有 DOM**，用 `display:none` 藏；**切换快**，一开始就建 DOM。
- **口诀**：**少切不展示用 if，老切显隐用 show**；只有 **v-if** 能配合 `v-else`。

**面试一句：** v-if 假的不进 DOM；v-show 一直在 DOM 里用 CSS 藏。频繁切换用 v-show、不常切换或初始不展示用 v-if。

### 2、表单输入（背诵版）

「在 Vue 里处理表单，**核心是用 `v-model` 做双向绑定**，同时要对应到具体控件：**单行输入**用 **`<input>`**，
**多行文本**用 **`<textarea>`**，**单选**用 **`<input type="radio">`**（同一组 radio 共用一个 `v-model`），
**复选**用 **`<input type="checkbox">`**（多选时 `v-model` 一般绑**数组**），**下拉**用 **`<select>`** 
配合里面的 **`<option>`**。**`v-model` 还有一些高级特性**，比如 **`.lazy`**（失焦再同步）、
还有 **`.number`、`.trim`** 等修饰符。表单**提交**一般在 **`<form>`** 上用 **`@submit.prevent`**，
**阻止浏览器默认提交**和整页刷新，然后在 **methods** 里自己处理，比如调接口。
**校验**可以在提交方法里先判必填；特别复杂的表单可以接 **VeeValidate** 或 **Element Plus** 这类方案。
若要自己做防抖、格式化，再拆成 **`:value` + `@input`**。」
