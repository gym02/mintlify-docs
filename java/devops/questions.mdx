## 七、DevOps

### 1、常见 Linux 命令（面试常用）

- 目录/文件：pwd，ls -al，cd，mkdir -p，touch，cp，mv，rm -rf  
- 查看文件：cat，less，head，tail -f  
- 查找/过滤：grep -n "xxx" file，find /path -name "*.log"，awk，sed  
- 权限：chmod，chown  
- 进程：ps -ef | grep xxx，top，kill -9 pid  
- 网络：ping，curl，wget，netstat -tunlp，ss -lntp  
- 磁盘/内存：df -h，du -sh *，free -h  
- 压缩：tar -czvf a.tar.gz dir，tar -xzvf a.tar.gz  
- 日志排组合：tail -f app.log | grep "ERROR"

### 2、常见 Docker 命令（面试常用）

- 镜像：docker images，docker pull xxx，docker rmi imageId  
- 容器：docker ps -a，docker run -d -p 8080:80 --name app image  
- 进入容器：docker exec -it app bash  
- 日志：docker logs -f --tail 200 app  
- 启停/删除：docker start/stop/restart app，docker rm -f app  
- 构建：docker build -t myapp:1.0 .  
- 推送：docker push myapp:1.0（需先 docker login，并且镜像名带仓库前缀，如 my-registry.com/ns/myapp:1.0）  
- 资源查看：docker stats  
- 清理：docker system prune  
- Compose：docker compose up -d，docker compose ps，docker compose logs -f，docker compose down  

### 3、Git 常用命令（面试常用）

- 初始化：git init  
- 克隆：git clone &lt;url&gt;  
- 查看状态：git status  
- 查看差异：git diff  
- 添加暂存区：git add . / git add &lt;file&gt;  
- 提交：git commit -m "msg"  
- 查看提交记录：git log --oneline  
- 查看分支：git branch  
- 新建并切换分支：git checkout -b &lt;branch&gt;（或 git switch -c &lt;branch&gt;）  
- 切换分支：git checkout &lt;branch&gt;（或 git switch &lt;branch&gt;）  
- 合并分支：git merge &lt;branch&gt;  
- 拉取：git pull  
- 推送：git push  
- 远程仓库：git remote -v  
- 暂存当前修改：git stash / git stash pop  
- 回退（未提交）：git restore &lt;file&gt;（或 git checkout -- &lt;file&gt;）  
- 回退（已提交）：git reset --soft/--mixed/--hard &lt;commit&gt;  
- 解决冲突：先手动改冲突文件，再 git add，再 git commit  

### 4、Nginx 反向代理和负载均衡怎么配置？（面试口述版）

**反向代理配置：**  
Nginx 收到请求后，不直接返回本地资源，而是把请求转发到后端服务器，并把后端响应再返回给客户端，对客户端来说只和 Nginx 打交道，这就是反向代理。核心用 `proxy_pass` 指定后端地址：

```nginx
server {
    listen 80;
    server_name www.example.com;
    location / {
        proxy_pass http://127.0.0.1:8080;   # 转发到本机 8080
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

常用搭配：`proxy_set_header` 把真实 Host、客户端 IP 等传给后端，便于后端做日志和鉴权。

**负载均衡配置：**  
在反向代理基础上，用 `upstream` 定义**多台**后端，Nginx 按策略选一台转发，就是负载均衡。`proxy_pass` 指向 upstream 名即可：

```nginx
upstream backend {
    # 调度策略（可选）：默认轮询；或 weight=2、ip_hash、least_conn 等
    server 192.168.1.10:8080 weight=3;
    server 192.168.1.11:8080 weight=1;
    server 192.168.1.12:8080 backup;   # 其他都不可用时才用
}
server {
    listen 80;
    location / {
        proxy_pass http://backend;   # 指向 upstream，实现负载均衡
    }
}
```

**常见调度方式：**  
- **轮询（round-robin）**：请求像发牌一样轮流分到各后端，是默认策略，适合机器性能相近、请求耗时差不多的场景。  
- **加权轮询（weight）**：在轮询基础上给高配机器更高权重，让它们多分一点流量，适合不同机器性能不一致的场景。  
- **ip_hash**：按客户端 IP 做哈希，同一 IP 固定落到同一台机器上，用于简单的会话保持（Session 在本机内存里时常用）。  
- **least_conn**（最少连接）：每次优先把请求发给“当前连接数最少”的后端，适合请求耗时差异比较大的场景，能动态把流量往相对空闲的机器上倾斜。

**面试可一句话概括：** 反向代理用 `proxy_pass` 指到一台后端；负载均衡先用 `upstream` 配多台 server 和调度策略，再 `proxy_pass http://upstream名` 即可。
